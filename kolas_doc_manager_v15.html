Google Drive 연동 설정

① Google Cloud Console에서 OAuth 2.0 클라이언트 ID(웹 애플리케이션)를 생성하고, 승인된 자바스크립트 원본에 이 페이지 접속 주소를 등록하세요.
② 스코프는 drive.file(앱이 생성한 파일만 접근)만 사용하므로 기존 Drive 파일에는 접근하지 않습니다.
③ 최초 로그인 시 Drive에 지정한 폴더가 자동 생성되고, 그 안에 metadata.json(문서대장)과 한글파일들이 저장됩니다.
④ 여기서 저장하면 이 브라우저에만 적용됩니다. 팀원 전체에게 자동 적용하려면, 관리자가 파일 상단의 DEFAULT_GOOGLE_CLIENT_ID 코드에 값을 직접 입력한 뒤 다시 배포하세요. Client ID는 비밀값이 아니라 공개되어도 안전합니다.

PDF 변환 설정 (CloudConvert)

① cloudconvert.com/dashboard/api/v2/keys에서 무료 계정으로 API 키를 발급받아 붙여넣고 "저장" 버튼을 눌러주세요.
② 등록된 한글(HWP/HWPX) 파일을 표 목록의 PDF 아이콘 버튼으로 클릭하면 CloudConvert를 통해 PDF로 변환되어 자동 다운로드됩니다.
③ 무료 티어는 매일 변환 크레딧이 제한되어 있습니다. 이 키 역시 이 브라우저에만 저장되며, 팀 전체 적용은 관리자가 배포 시 값을 넣어야 합니다.

Firebase 로그인 서버 설정

Firebase 콘솔(console.firebase.google.com)에서 프로젝트를 만들고 Authentication → Sign-in method에서 "이메일/비밀번호"를 사용 설정하세요. 팀원 계정은 Authentication → Users 탭에서 관리자가 직접 추가/삭제합니다.
여기서 저장하면 이 브라우저에만 적용됩니다. 팀원 전체에게 자동 적용하려면, 관리자가 파일 상단의 DEFAULT_FB_CONFIG 코드에 값을 직접 입력한 뒤 다시 배포하세요. apiKey/authDomain 등도 공개되어도 안전한 값입니다(보안은 Firebase 서버 규칙으로 보호됩니다).

사용자 권한 관리

권한 등급 관리는 관리자 계정만 가능합니다. 본인 권한은 아래 목록에서 확인할 수 있습니다.
관리자: 등록·수정·삭제 모두 가능  /  편집자: 등록·수정 가능(삭제 불가)  /  조회자: 열람·다운로드만 가능(등록·수정·삭제 불가).
계정은 Firebase 로그인 이메일 기준이며, 이 목록은 Drive 폴더의 roles.json에 저장되어 같은 문서대장(Drive 폴더)을 쓰는 팀원 전체에 공통 적용됩니다.
최초로 Google Drive에 연결한 계정은 자동으로 관리자로 등록됩니다.
